    Like a few people I broke my data up into a bunch of parts, and figured out the ngrams for each part. Then, for each part file I calculated the frequencies of each ngram, which gave me how likely that ngram was to come up. But at this point I need to bring all my individual frequencies together to get how likely frequencies are overall. One way is rbinding the individual frequencies then aggregating an overall sum, another way is using successive merges and adding each column. Both of these are punishingly slow. But...

    Both of the previous two options need to allow for the possibility of there being many matching entries (either 1 to many or aggregating many) but given in this case each file is a file of the frequency, each key is unique to the file and will be present at most once per file. We can take significant advantage of this fact. consider the following:

    version1 <- function (){
a <- data.frame(k=1:1000000, v=rep(1,1000000))
b <- data.frame(k=500001:1500000, v=rep(1,1000000))
c <- merge(a,b, by="k", all=TRUE)
c$v.x[is.na(c$v.x)] <- 0
c$v.y[is.na(c$v.y)] <- 0
c$v <- c$v.x + c$v.y
c$v.x <- NULL
c$v.y <- NULL
c
}

version2 <- function (){
  a <- data.frame(k=1:1000000, v=rep(1,1000000))
  b <- data.frame(k=500001:1500000, v=rep(1,1000000))
  c <- rbind(a,b)
  d <- tapply(c$v,c$k,sum)
  d
}

version3 <- function (){
a <- data.frame(k=1:1000000, v=rep(1,1000000))
b <- data.frame(k=500001:1500000, v=rep(1,1000000))
a <- a[order(a$k),]
b <- b[order(b$k),]
a$v[a$k %in% b$k] <-  a$v[a$k %in% b$k] + b$v[b$k %in% a$k]
c <- rbind(a, b[!(b$k %in% a$k),])
c
}

system.time(version1())
system.time(version2())
system.time(version3())

    Maher HarbSignature Track· 5 days ago 
    ~20 times faster than version3 using data.table!

    version4 <- function (){
  library(data.table)
  a <- data.table(k=1:1000000, v=rep(1,1000000))
  b <- data.table(k=500001:1500000, v=rep(1,1000000))
  c <- rbind(a,b)
  c <- c[, lapply(.SD, sum), by = c("k")]
  c
}

    system.time(version4())

    David HoodCommunity TA· 4 days ago 

    My best time to beat at the moment for actual course files is coming from Python using dictionaries to accumulate the frequency results from individual ngram files (Though I haven't played with the latest version4 Maher came up with during my nighttime). This (working with 70% of the data) is actually pretty speedy for both accumulating the totals, and figuring out what the most likely predictor is and saving the ones that are not the most likely into another file. I should add I am using python 2.7 on a unicode native system, so do not need to worry about utf-8 conversion issues when reading or writing the files (while python is pretty good (and Python 3 is better), I find it is not as good as R).

    Speed-wise this works so well because of its dependance on O(1) hashes for all the lookup operations, so the time is reading in the files, and iterating through a combined list three times writing out files.

    Imagine the starting files are rows of gram1 gram2 gram3 prediction frequencyWithinFile

    cumlativeEntries = {}
maxcumlative = {}
for letters in ("tw","bl","nw"):
    for i in range(70):
        filepath = "n4Freq/" + str(i+1).zfill(3) + letters + ".txt"
        with open(filepath,'r') as f:
            for line in f:
                bits = line.split(" ")
                keybit = bits[0] + " " + bits[1] + " " + bits[2] + " " + bits[3]
                valbit = int(bits[4])
                if (keybit in cumlativeEntries):
                    cumlativeEntries[keybit] = cumlativeEntries[keybit] + valbit
                else:
                    cumlativeEntries[keybit] = valbit

with open('n4cumulfreq/totals.txt','w') as f:
    for key, value in cumlativeEntries.iteritems():
        f.write(key + " " + str(value) + "\n")
        keybits = key.split(" ")
        key3 = keybits[0] + " " + keybits[1] + " " + keybits[2]
        if (key3 in maxcumlative):
            if (value > maxcumlative[key3]): maxcumlative[key3] = value
        else:
            maxcumlative[key3] = value

with open('n4cumulfreq/maxedtotals.txt','w') as f:
    for key, value in cumlativeEntries.iteritems():
        keybits = key.split(" ")
        key3 = keybits[0] + " " + keybits[1] + " " + keybits[2]
        best = maxcumlative[key3]
        if (value == best) : f.write(key + " " + str(value) + "\n")

with open('n4cumulfreq/excludedtotals.txt','w') as f:
    for key, value in cumlativeEntries.iteritems():
        keybits = key.split(" ")
        key3 = keybits[0] + " " + keybits[1] + " " + keybits[2]
        best = maxcumlative[key3]
        if (value < best) : f.write(key + " " + str(value) + "\n")
