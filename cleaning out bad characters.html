    After reading your comment I remembered that strsplit can be used to bust up strings into characters, even the ones that are UTF-8.  So, I wrote a short script to compute the character counts for all three files and line them up in a table, so I could see which "odd" characters appear in which file:

    ...

    charCounts <- function(strings, name)
    {
      counts <- data.frame(table(char=unlist(strsplit(strings,""))))
      counts$char <- as.character(counts$char)
      names(counts)[2] <- name
      counts
    }       

    counts.blogs   <- charCounts(blogs,   "blogs")
    counts.news    <- charCounts(news,    "news")
    counts.twitter <- charCounts(twitter, "twitter")

    counts.all <- merge( merge(counts.blogs, counts.news, all=TRUE), counts.twitter, all=TRUE)    

    # Add TOTAL row
    counts.all <- rbind(counts.all, c("TOTAL", apply(counts.all[,-1], 2, sum, na.rm=TRUE)))                      

     

    How many different UTF-8 characters are you seeing?  In all three files, I'm seeing about 212 different characters in the over half billion characters in all three files.

    [The en_US.blogs.txt file has 210,160,014 bytes, but my processing through strsplit resulted in 208,361,438 characters -- so there were a number of multi-byte UTF-8 characters.]

    I computed a rate column showing the number of occurrences of each character per million characters.  I'm seeing nearly 52 million character "e" in all three files -- or about 90,400 times per million characters.  An "E" occurs 746,202 times or about 1300 times per million characters. 

    But the 7 variations of accented e's/E's in total only occur about 2 times per million characters.  These stats tell me it's likely not worth bothering with the accented non-English "E" characters in the three files we're working with -- I'll treat them all as E's (with apologies to the French).  Are you seeing similar stats?
